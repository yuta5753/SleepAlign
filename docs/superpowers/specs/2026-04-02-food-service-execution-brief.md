## 1. プロジェクト概要

### サービス名（仮）
**飲食店経営リモデリング支援**

### 一言で言うと
3〜10店舗の飲食企業に対し、AI活用を前提に業務フロー・コスト構造・人員配置を「構造ごと再設計」するプロジェクト型コンサルティング。

### 差別化の核
**「AIで何を自動化するか」ではなく、「AIがある前提で会社をどう作り直すか」**

### 3層コンセプト
| 層 | 内容 | 役割 |
|---|---|---|
| コンセプト（本質） | 経営構造のリモデリング | 提供価値の本体 |
| 入口（フック） | コスト構造の可視化・削減 | 興味を引く入口 |
| 手段（How） | AIの活用 | 手段としてのAI |

### サービス構成

| フェーズ | 名称 | 内容 | 期間 | モニター価格 | 通常価格 |
|---------|------|------|------|------------|---------|
| Phase 1 | 現状の可視化 | 業務フロー・コスト構造・人員配置の棚卸しと分析 | 1ヶ月 | 10〜15万円 | 30〜50万円 |
| Phase 2 | 構造の再設計 | AI活用を前提とした業務・収益・組織のリモデリング設計 | 1〜2ヶ月 | （フルパッケージ） | （フルパッケージ） |
| Phase 3 | 実装・定着支援 | 設計した仕組みの導入と運用サポート | 1〜2ヶ月 | 50万円前後 | 100〜300万円 |

### ポジショニング

| | SaaS・ツール導入 | **本サービス** | 大手コンサル |
|---|---|---|---|
| やること | 個別業務の効率化 | **経営構造の再設計** | 経営全般の改革 |
| 価格帯 | 月数千〜数万円 | **50〜300万円** | 500万円〜 |
| 対象規模 | 全規模 | **3〜10店舗** | 10店舗以上 |

## 2. ターゲット

### メインターゲット
**3〜10店舗の飲食企業の経営者・オーナー**

| 項目 | 内容 |
|------|------|
| 年商イメージ | 1億〜10億円前後 |
| 組織の状態 | 社長+店長+現場スタッフ。本部機能は薄い or 社長が兼務 |
| 典型的な悩み | 「自分がいないと回らない」「店舗が増えるほど管理が追いつかない」 |
| AIへの認識 | 興味はあるが何から始めていいかわからない |

### ターゲットの心理
- 「なんとなく大変」だが、何が問題かを数字で把握できていない
- ツールを入れたが「やることが増えただけ」と感じている
- 店舗を増やしたいが、今の体制では限界を感じている
- AI・DXは気になるが、自社に合うのかわからない

## 3. メッセージング指針

### 入口のフック（場面別）

| パターン | フック | 使用場面 |
|---------|-------|---------|
| 壁フック | 「3店舗の壁、感じていませんか？」 | セミナータイトル・SNS・資料の見出し |
| コストフック | 「社長の時間、月何時間バックオフィスに消えていますか？」 | DM・個別面談・提案書 |
| 構造フック | 「AIを入れても楽にならなかった理由、わかりますか？」 | note記事・コンテンツ |

### キーメッセージ集
- 「ツールを入れても楽にならない」のは、業務フローと役割が昔のままだから
- 「3店舗の壁」「5店舗の壁」— 90%の企業が3〜5店舗で停滞する
- AI活用の本質は「何を自動化するか」ではなく「会社をどう作り直すか」
- まず見える化から。数字で見えれば、打ち手が変わる
- Phase 1だけでも価値がある。その結果を見て次を判断してもらえる

### 「リモデリング」の扱い
- **対面・口頭**: 「会社の仕組みの作り直し」「経営の土台の再設計」と平易に言い換え
- **資料上**: 「経営構造リモデリング」としてブランドワードとして使用
- **セミナー**: 「3店舗の壁を超える経営構造改革」のように具体課題と結びつける

### 禁止表現
- 「絶対」「必ず」「劇的に」「革命的」等の誇大表現
- 「業界初」「唯一の」等の裏付けなき最上級表現
- 「あなたの店はダメ」等の否定表現
- 「私が教えてあげる」的な上から目線
- AIへの過度な期待を煽る表現（「AIで全部解決」等）

### トーンの基準
- **現場を知る専門家のフラットな語り口**
- 飲食業界の構造を理解している人が、同じ目線で話している感覚
- データ・事実に基づいた論理的な説明
- 「です・ます」ベース。堅すぎず、軽すぎず

## 4. ブランド・デザイン指針

### ビジュアルの方向性
**信頼×実務的** — データや図解を多用し、専門家としての信頼感を重視。大手コンサルほど堅くないが、スタートアップ的なカジュアルさは避ける。

### カラーパレット

| 用途 | 色 | コード | 使い方 |
|------|---|--------|--------|
| メインカラー | ネイビー | `#1a365d` | 見出し・ヘッダー・枠線 |
| サブカラー | ダークグレー | `#374151` | 本文テキスト |
| アクセント | ゴールド/アンバー | `#d97706` | 強調・CTA・モニター特典 |
| 背景（メイン） | ホワイト | `#ffffff` | ページ背景 |
| 背景（セカンダリ） | ライトグレー | `#f8fafc` | カード・テーブル背景 |
| 背景（強調ブロック） | ライトブルー | `#eff6ff` | 成果物・ポイント強調 |
| 背景（警告・課題） | ライトアンバー | `#fef3c7` | 課題提示ブロック |

### タイポグラフィ
- **フォント**: Hiragino Kaku Gothic ProN / Noto Sans JP（ゴシック体）
- **見出し**: 太め（700）、ネイビー
- **本文**: レギュラー（400）、ダークグレー、12〜13px相当
- **階層**: タイトル > セクション見出し > 本文 > 注釈 の4段階を明確に

### レイアウトの原則
- **余白を十分に取る** — 詰め込まない。1ページ1メッセージ
- **数字は大きく見せる** — 統計データ・金額は視覚的に目立たせる
- **図解・テーブルを多用** — 文章の壁を避ける
- **3色以内** — ネイビー + グレー + アクセント（ゴールド）

### 資料品質の基準
- ヘッダー・タイトルの見切れなし
- 意図したページ数に収まっている
- テーブルがページ境界で不自然に分割されていない
- フォントサイズが読みやすい（最小10px相当）
- 印刷しても視認性が保たれる

## 5. 資材制作リスト

### 最優先（Track A: モニター獲得用）

#### 5-1. アプローチ文面（DM/メール）
- **目的**: 既存ネットワークへのモニター声がけ
- **形式**: テキスト（Markdown）。カジュアルDM版 + フォーマルメール版の2パターン
- **要件**: カスタマイズ可能な【 】付きテンプレート。チェックリスト付き
- **ステータス**: 初版作成済み（`~/Desktop/claude-outputs/04_店舗ビジネス支援/飲食リモデリング/営業ツール/01_アプローチ文面.md`）→ 品質確認・改善の余地あり

#### 5-2. 課題ヒアリングシート
- **目的**: 初回面談（60分）の進行ガイド兼メモシート
- **形式**: PDF（A4・2〜3ページ）。面談時に手元で使う想定
- **構成**: 基本情報 → 社長の1週間 → ツール状況 → 課題の深掘り（SPIN型） → 次のステップ
- **要件**: 記入欄あり。印刷して使える体裁。面談後のアクションリスト付き
- **ステータス**: Markdown初版あり（`02_課題ヒアリングシート.md`）→ **PDF化・ビジュアル整備が必要**

#### 5-3. Phase 1提案書
- **目的**: モニター提案時に渡す1枚もの
- **形式**: PDF（A4・1ページ）
- **構成**: 課題提示 → サービス概要（3フェーズ） → Phase 1成果物 → 実施概要 → モニター価格
- **要件**: 渡した瞬間に「何をしてくれるのか」がわかるビジュアル。印刷対応
- **ステータス**: HTML/PDF初版あり（`03_Phase1提案書.pdf`）→ **品質改善が必要**（レイアウト・情報の優先度・視覚的インパクトを再設計）

### 高優先（Track A補完 + Track B準備）

#### 5-4. サービス概要資料（3〜5ページ）
- **目的**: サービスの詳細説明。面談後やセミナー後に送付
- **形式**: PDF（A4・3〜5ページ）
- **構成案**:
  1. 表紙（サービス名 + キーメッセージ）
  2. 課題提示（「3・5・10店舗の壁」+ データ）
  3. サービス概要（3フェーズの全体像 + 各フェーズの成果物）
  4. 競合との違い（ポジショニングマップ）
  5. 価格・モニター案内・次のステップ
- **要件**: 見出しだけ読んでもストーリーが通る。データ・図解を多用
- **ステータス**: 未着手

#### 5-5. セミナースライド
- **目的**: セミナー「3店舗の壁を超える — AI時代の飲食店経営の仕組みづくり」用
- **形式**: PDF or HTML（16:9・30〜40枚想定）
- **構成**:
  1. 導入: 「3・5・10店舗の壁」の解説（15分）
  2. 問題提起: なぜツールを入れても楽にならないのか（15分）
  3. 解決の方向性: 「構造ごと作り直す」という考え方（20分）
  4. 事例: AI活用の具体例（20分）
  5. まとめ・個別相談への誘導（20分）
- **要件**: 1スライド1メッセージ。文字数100〜200字/スライド。セミナー資料ルール準拠
- **ステータス**: 未着手

### 中優先（集客施策）

#### 5-6. 自己診断ツール（属人化度チェックリスト）
- **目的**: セミナー内で使用 or SNSで配布。「自社の状態に気づく」体験を提供
- **形式**: PDF（A4・1ページ）or Webフォーム
- **構成案**: 10〜15問のYes/No質問 → スコアリング → 3段階の結果表示
- **質問の軸**: 社長依存度 / 業務の標準化度 / ツール活用度 / 人員配置の柔軟性
- **要件**: 1分で回答できる手軽さ。結果が「もっと知りたい」につながる設計
- **ステータス**: 未着手

#### 5-7. note記事シリーズ（「壁」シリーズ）
- **目的**: 認知獲得・専門性の発信。SEO + SNS拡散
- **記事案**:
  - 「3店舗の壁」の正体 — 90%の飲食企業が停滞する構造的理由
  - 飲食店がAIを入れても楽にならない本当の理由
  - 社長の時間を取り戻す「構造の見える化」とは
- **要件**: noteブランドルール準拠（CLAUDE.md参照）
- **ステータス**: 未着手

## 6. 使用データ・エビデンス一覧

資材制作時に引用可能なデータを一覧化する。出典を必ず添えること。

### ターゲットの課題

| データ | 数値 | 出典 |
|--------|------|------|
| 3〜5店舗で停滞する企業の割合 | **90%** | 常進パートナーズ |
| 「特定の人しかわからない業務がある」 | **57.3%** | バックオフィス業務調査 |
| 「デジタル化されていない業務に時間がかかる」 | **47.3%** | 同上 |
| 「複数システムでデータがバラバラ」 | **43.6%** | 同上 |
| 非正社員の人手不足感（飲食業・全業種トップ） | **67.0%** | 帝国データバンク 2025年4月 |
| 飲食業の離職率 | **26.6%**（全産業平均15.4%の1.7倍） | 厚労省 令和5年雇用動向調査 |
| 飲食店倒産件数（2025年上半期・過去最多） | **458件** | 東京商工リサーチ |
| 営業利益率（黒字企業平均） | **8.6%**（小規模一般飲食店: 4.3%） | TKC経営指標 |
| FLコスト（食材費+人件費） | **55〜60%** | 業界一般 |

### 「3・5・10店舗の壁」

| 壁 | 発生する課題 | 根本原因 |
|----|------------|---------|
| 3店舗の壁 | 社長の目が届かなくなる | ビジョン共有の不足。社長と従業員の温度差 |
| 5店舗の壁 | 管理業務が爆発的に増加 | 仕組み・システムの不在。本部機能が必要 |
| 10店舗の壁 | 人・物・金すべての標準化が必須 | 評価・給与・教育体系など組織制度の不在 |

> 出典: 常進パートナーズ / 中西フードビジネス研究所

### 競合・市場

| データ | 内容 |
|--------|------|
| 競合の空白地帯 | 「AI×経営構造設計×3〜10店舗」にプレイヤーなし |
| プロジェクト制コンサル相場 | 50万〜500万円 |
| 月額コンサル相場（中小向け） | 月額10万〜80万円 |
| デジタル化・AI導入補助金2026 | 補助率1/2〜4/5、上限最大450万円程度 |
