## 1. サービスコンセプト

### サービス名（仮）
**飲食店経営リモデリング支援**

### コンセプト文
エージェントAI時代の到来を前提に、飲食企業の業務フロー・収益構造・人員配置を「今の延長線上の改善」ではなく「構造ごと再設計」するプロジェクト型支援サービス。

### コンセプトの3層構造
- **コンセプト（本質）**: 経営構造のリモデリング
- **入口（フック）**: コスト構造の可視化・削減
- **手段（How）**: AIの活用

「コスト削減できますよ」で興味を引き → 「浮いたリソースで構造自体を変えませんか？」と本題に持っていく → 「その手段としてAIをこう使います」という流れ。

### 差別化のキーメッセージ
**「AIで何を自動化するか」ではなく、「AIがある前提で会社をどう作り直すか」**

### ポジショニング
| | SaaS・ツール導入 | **本サービス** | 大手コンサル |
|---|---|---|---|
| やること | 個別業務の効率化 | **経営構造の再設計** | 経営全般の改革 |
| 視点 | ツール単位（点） | **構造全体（面）** | 戦略全体（面） |
| 価格帯 | 月数千〜数万円 | **50〜300万円** | 500万円〜 |
| 対象規模 | 全規模 | **3〜10店舗** | 10店舗以上 |
| AI活用の捉え方 | ツールとして導入 | **構造変革の手段** | DX戦略の一部 |

## 2. ターゲット

### メインターゲット
**3〜10店舗の飲食企業の経営者・オーナー**

| 項目 | 内容 |
|------|------|
| 店舗数 | 3〜10店舗 |
| 年商イメージ | 1億〜10億円前後 |
| 組織の状態 | 社長+店長+現場スタッフ。本部機能は薄い or 社長が兼務 |
| 典型的な悩み | 「自分がいないと回らない」「店舗が増えるほど管理が追いつかない」 |
| AIへの認識 | 興味はあるが何から始めていいかわからない / ツールを入れてみたが使いこなせていない |

### サブターゲット（ライト版）
**個人店〜3店舗のオーナー**
- Phase 1の内容をテンプレート化・簡易化したもの
- 将来的にはオンラインコンテンツ化（セミナー動画+テンプレ提供）の可能性あり

### ターゲットが抱える構造的課題（リサーチ検証済み）

飲食コンサルタントの調査では、**相談企業の90%が3〜5店舗で停滞**し、10%の企業だけが10店舗以上に拡大できている。この停滞の根本原因は「人の問題」と「仕組みの問題」の2つに集約される。

**1. 「3店舗・5店舗・10店舗の壁」— 段階的に発生する構造課題**

| 壁 | 発生する課題 | 根本原因 |
|----|------------|---------|
| 3店舗の壁 | 社長の目が届かなくなる。創業メンバー以外の店長が育っていない | 社長と従業員の温度感のギャップ。ビジョン共有の不足 |
| 5店舗の壁 | 管理業務が爆発的に増加。個人の能力では管理しきれない | 仕組み・システムの不在。本部機能が必要だがコストをかけられない |
| 10店舗の壁 | 人・物・金すべての管理手法の標準化が必須 | 評価・給与・教育体系など組織制度の不在 |

> 出典: [常進パートナーズ](https://johshin.co.jp/blog/1778) / [中西フードビジネス研究所](https://note.com/quick_elk9771/n/n7c2eec5eb526)

**2. 属人化とデジタル化の遅れ — 業界全体の構造問題**
- バックオフィス業務課題の**トップは「特定の人しかわからない業務がある」（57.3%）**
- 「デジタル化されていない業務に時間がかかる」（47.3%）
- 「複数システムでデータがバラバラに分散」（43.6%）
- ツールを入れても業務フローが変わらないため「やることが増えただけ」になっている

**3. 人手不足 — 全業種で最も深刻**
- 非正社員の人手不足を感じている企業は**67.0%で全業種トップ**（3年連続）（帝国データバンク 2025年4月）
- 離職率**26.6%**（全産業平均15.4%の1.7倍）（厚労省 令和5年雇用動向調査）
- 人手不足により営業時間短縮を余儀なくされる店舗が増加
- 飲食店倒産件数は2025年上半期に**過去最多の458件**（3年連続増加）

**4. 利益率の圧迫 — 投資余力の不足**
- 営業利益率は全体平均8.6%、小規模一般飲食店で**4.3%**（黒字企業平均）
- 人件費率の目安は売上の約30%、FLコスト（食材費+人件費）は55〜60%
- 食材費・光熱費の高騰で利益率がさらに低下傾向

## 3. サービス構成

### プロジェクト全体像（3フェーズ構成）

| フェーズ | 名称（仮） | 内容 | 期間目安 |
|----------|-----------|------|----------|
| Phase 1 | 現状の可視化 | 業務フロー・コスト構造・人員配置の棚卸しと分析 | 1ヶ月 |
| Phase 2 | 構造の再設計 | AI活用を前提とした業務・収益・組織のリモデリング設計 | 1〜2ヶ月 |
| Phase 3 | 実装・定着支援 | 設計した仕組みの導入と運用サポート | 1〜2ヶ月 |

### 各フェーズの成果物

**Phase 1：現状の可視化**（入口のフック＝コスト構造の可視化）
- 業務フローマップ（誰が・何を・どのくらいの時間でやっているか）
- コスト構造分析シート（人件費・外注費・ツール費の内訳と「AI代替可能性」の判定）
- 課題マップ（構造的なボトルネックの特定）

**Phase 2：構造の再設計**（サービスの核）
- 新業務フロー設計（AI活用後の「あるべき姿」）
- 人員・役割の再定義（人がやること / AIがやることの切り分け）
- 収益構造シミュレーション（コスト削減効果＋再投資先の設計）
- 実行手段の選定（下記参照）
- （必要に応じて）評価・給与体系の改定案

**Phase 2における実行手段の選定：**

| 課題に対する解決手段 | 対応 |
|---------------------|------|
| 業務フロー・人員配置の変更 | 辻が設計・支援 |
| AIエージェント活用 | 辻が選定・導入設計 |
| 軽量なシステム開発 | バイブコーディングで対応 |
| 本格的なシステム開発 | 外部パートナーに繋ぐ（辻は要件定義・設計の立場） |

**Phase 3：実装・定着支援**
- AI活用ツール・運用の導入サポート
- スタッフ向けオペレーション変更の落とし込み
- 定着フォロー（1〜2ヶ月の伴走）

### 価格構成

| | モニター価格 | 通常価格（実績後） |
|---|---|---|
| Phase 1のみ（ライト版） | 10〜15万円 | 30〜50万円 |
| Phase 1〜3 フルパッケージ | 50万円前後 | 100〜300万円 |

- Phase 1だけ受けて「あとは自分でやります」もOK → 入口のハードルを下げる
- Phase 1で現状が見えた結果、Phase 2以降に自然につながる設計

**価格の妥当性（リサーチ検証済み）:**
- プロジェクト制コンサルの相場は50万〜500万円 → 本サービスはこの中間帯で妥当
- 3店舗（年商1億円）にはPhase 1単独（10〜15万円）から入るべき
- 10店舗（年商5億円以上）にはフルパッケージ300万円も現実的
- 月額換算で見せる選択肢も有効（例：100万円÷4ヶ月＝月額25万円。月額コンサル相場の範囲内）

**補助金活用による投資ハードル低減:**
- **デジタル化・AI導入補助金2026**（旧IT導入補助金）：Phase 3で導入するAIツール・システムが補助対象になる可能性が高い
- 補助率1/2〜4/5（小規模事業者は最大80%）、補助上限は最大450万円程度
- 補助金申請支援を付加サービスとして提供するとさらに有効
- 他に事業再構築補助金、小規模事業者持続化補助金、業務改善助成金も活用可能性あり

## 4. 差別化と競合ポジショニング

### 競合比較（リサーチ検証済み）

| 観点 | SaaS・ツール導入 | IT企業のAI導入支援 | 大手コンサル | **本サービス** |
|------|-----------------|-------------------|-------------|---------------|
| 売り文句 | 「この業務が楽になります」 | 「AIを導入しましょう」 | 「経営を変革します」 | **「会社の構造を作り直します」** |
| アプローチ | ツール単体の導入 | 技術起点 | 戦略起点 | **構造設計起点** |
| AI活用の位置づけ | ツールそのもの | 導入が目的化しがち | DX戦略の一要素 | **再設計の手段** |
| 価格帯 | 月額数千〜数万円 | 数十万〜数百万円 | 月額10万〜80万円（年間120万〜960万円） | **50〜300万円（プロジェクト型）** |
| 主要プレイヤー | スマレジ、トレタ、Airレジ、freee等 | EBILAB、デリシャスノーツ等 | 船井総研、日本経営等 | **（新規参入）** |
| 課題 | 入れても構造が変わらない | 経営の文脈がわからない | 高すぎる・重すぎる | **実績をこれから作る** |
| 相性が良い企業規模 | 全規模 | 中〜大規模 | 中堅〜大規模 | **3〜10店舗** |

**リサーチで確認された競合の空白地帯：**
- 「AI×経営構造設計」を3〜10店舗向けに提供するプレイヤーは**見当たらない**
- コンサルは「売上改善・メニュー開発」が中心で構造設計まで踏み込まない
- DX・AI導入支援は技術起点で業務フロー・人員配置・評価制度には踏み込まない
- SaaSベンダーはツールを売るが、ツールが機能する土台（業務フロー・組織設計）は提供しない

### 本サービスが勝てるロジック
**「ツールを入れたのに楽にならない」問題を解決する立ち位置**

業務フロー・人の役割・評価制度が昔のままなので、ツールを入れても「やることが増えただけ」になっている。本サービスは「ツールが活きる土台を作る存在」。ツールベンダーの敵ではなく、ツールの効果を最大化する上流の仕事。

### 辻の強みが活きるポイント

| 強み | サービスでの活かし方 |
|------|-------------------|
| 組織人事コンサルの実績 | 人員配置・役割再定義・評価制度の設計はそのまま使える |
| 飲食業界の実績 | 業界特有の構造（シフト・原価・店舗運営）を理解している |
| 自身がAIヘビーユーザー | 「自分がやってみて成果を出した」が説得力になる |
| 店舗ビジネスの仕組み構築 | エステ・小売での仕組み設計の経験を転用できる |

### 認知の再定義
現状「教育・評価制度の人」として認知されている → 本サービスを通じて「仕組み×AIの人」に認知を上書きする。

### リスクと対策（リサーチ検証反映）
- **「AI×飲食×構造改革」の実績がまだない** → モニター案件で早期に作る（方針変更なし）
- **「リモデリング」が抽象的で伝わりにくい可能性** → Phase 1の「可視化」を入口にして具体性を持たせる。「3店舗の壁」「5店舗の壁」という共感しやすい言葉でフックを作る（営業戦略フェーズで検証）
- **ターゲット企業がこの課題を認識しているか** → リサーチの結果、属人化（57.3%）・人手不足（67.0%）は認識されている。ただし「構造を変える」という解決策への認知は薄い → 「コスト可視化」で入り「構造変革」に導く設計は有効
- **競合参入リスク** → 現時点で「AI×経営構造設計×3〜10店舗」は空白地帯。ただし船井総研がAI活用セミナーを開始しており、大手の参入可能性あり → 早期に実績・事例を確保することが重要
- **価格に対する抵抗感** → 営業利益率4〜8%の中での50万〜300万円は大きな投資判断。補助金活用提案と段階的フェーズ構成で対応

## 5. サービス提供プロセスとクライアント体験

### クライアント体験フロー

```
認知・興味
  ↓ 「コスト構造、見えてますか？」というフックで関心を引く
初回接点（セミナー or 個別相談）
  ↓ 現状の構造的な問題に「気づく」体験を提供
Phase 1 受注（可視化）
  ↓ 自社の業務・コスト・人員の現状が初めて「見える」
分析結果の報告
  ↓ 「ここ、構造ごと変えたほうがいいですね」という提案
Phase 2 受注（再設計）
  ↓ AIを前提にした新しい構造の設計
Phase 3 受注（実装・定着）
  ↓ 設計を現場に落とし込む
継続関係
  ↓ 定期的な構造レビュー / 新たな課題への対応
```

### Phase 1→2の自然な接続（設計上の最重要ポイント）
- Phase 1で数字として可視化 → 「なんとなく大変」が「ここに月○万円」に変わる
- その数字を見た上で「AIで代替できるが、業務フローと役割の変更が必要」と提示
- 「コスト削減」で入って「構造改革」に自然に転換する仕掛け

### コミュニケーション設計

| 項目 | 内容 |
|------|------|
| 定例ミーティング | 月2回程度（Phase 1は現場ヒアリング含む） |
| 成果物の共有 | 各フェーズ終了時にレポート・設計書を納品 |
| 意思決定タイミング | Phase 1終了時に「Phase 2に進むか」を判断（強引に売らない設計） |
| 経営者の関与度 | Phase 1は経営者+現場責任者、Phase 2以降は経営者中心 |

### モニター期間の特別設計
- モニター価格の代わりに得るもの：事例掲載許可・アンケート回答・推薦コメント
- プロセス自体を記録：ビフォーアフターのデータを蓄積し、営業資材にする
- 過剰サービスにしない：Phase構成は同じ、価格だけ下げる

|--------|------|------|

| **高** | ③ 人員配置・役割の再定義 | 人手不足67.0%で全業種最悪。「人がやること/AIがやること」の切り分けが急務 |

> Phase 1の入口は「②収益構造の可視化」で興味を引き、Phase 2で「①業務フロー再設計」「③人員配置再定義」「⑤バックオフィスAI化」を中心に構造を変える、という流れが最も自然。

## 7. 営業戦略

### 2トラック営業戦略

モニター獲得（短期）とセミナー営業（中期）の2トラックを並走させる。

**Track A: モニター獲得（今すぐ〜2026年5月頭）**

| ステップ | 内容 | 期限 |
|---------|------|------|
| 1. ターゲットリスト作成 | 既存ネットワークから3〜5社を選定 | 4月第1週 |
| 2. アプローチ連絡 | DM or メールで声がけ | 4月第1〜2週 |
| 3. 課題ヒアリング面談 | 60分・対面。課題の深掘りと信頼構築 | 4月第2〜3週 |
| 4. Phase 1提案 | 面談の内容を踏まえて個別提案 | 面談後3日以内 |
| 5. モニター契約 | 条件提示・合意 | 5月頭まで |

- **目標**: 1ヶ月以内に1〜2社のモニター契約
- **選定基準**: 3〜10店舗 / 社長と直接話せる関係 / 「自分がいないと回らない」と感じていそう
- **モニター条件**: Phase 1を10〜15万円 / 事例掲載許可 / 推薦コメント / プロセスの記録許可

**Track B: セミナー起点の営業導線（並行準備〜2ヶ月目以降）**

```
セミナー参加（「気づき」を提供）
  ↓
個別相談（課題の具体化）
  ↓
```

### メッセージング戦略

**入口のフック（場面別に使い分け）:**

| パターン | フック | 想定場面 |
|---------|-------|---------|
| A: 壁フック | 「3店舗の壁、感じていませんか？」 | セミナータイトル・SNS |

**「リモデリング」の伝え方:**
- **対面（面談・商談）**: 「会社の仕組みの作り直し」「経営の土台の再設計」と平易に言い換え
- **セミナー**: 「3店舗の壁を超える経営構造改革」のように具体的な課題と結びつける
- **資料・コンテンツ**: 「リモデリング」をブランドワードとして使用し、認知を定着させる
- **検証方法**: モニター面談・セミナーでの反応を観察し、刺さる表現を特定する（設計ドキュメント#6の検証を兼ねる）

**営業トークの基本構造（SPIN型）:**

```
状況質問: 「今、何店舗ですか？ 次の出店は考えていますか？」
問題質問: 「社長がいない日、現場はどうなっていますか？」
示唆質問: 「もし社長が倒れたら、3店舗とも回りますか？」
解決質問: 「社長がいなくても回る仕組みを、AIを使って作れるとしたら？」
```
