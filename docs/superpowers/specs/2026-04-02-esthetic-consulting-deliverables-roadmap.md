## 各成果物の設計メモ

### 1. 見込み顧客への提案書

**ゴール**: ビジネスパートナーが初回面談後に渡す/送付する資料。「この人に頼みたい」と思わせる。

**構成案**（企画書セクション6より）:
1. 相手の状況への共感（ヒアリング内容を反映）
2. メインフック（「新規20人 vs 5人」）
3. ファクトで裏付け（リピート率21%・5倍コスト差）
4. サービスの流れ（5ステップ）
5. 他のコンサルとの違い
6. オプション紹介
7. 価格
8. 次のアクション（Step1への申込み）

**カスタマイズポイント**: 冒頭の「相手の状況への共感」と「期待される成果」を相手ごとに差し替える

**エージェントチーム案**: `/make-deck`スキル活用。リサーチ不要（既に完了）→ 構成設計 → 本文作成 → 図解 → デザイン → 最終編集

---

### 2. Webページ（LP/サービス紹介）

**ゴール**: オンラインで見込み客が「問い合わせしたい」と思うページ。

**構成案**（企画書セクション6より）:
1. ファーストビュー（メインフック + ひと言コンセプト）
2. 悩み共感セクション（「こんなお悩みありませんか？」）
3. ファクト提示（業界データで「実はこうなんです」）
4. サービス概要（5ステップを視覚的に）
5. 他のコンサルとの違い（比較表）
6. オプション紹介
7. CTA（Step1への申込み or 無料相談）

**注意点**:
- 専門用語（CX・LTV等）は使わない
- フックと語り方設計の言葉をそのまま使う
- バイブコーディングでの実装を想定

**エージェントチーム案**: 構成設計 → コピーライティング → HTML/CSS実装

---

### 3. サービス概要説明資料

**ゴール**: ビジネスパートナーが営業時に見せる/渡す資料。A4で2〜4ページ程度。

**構成案**:
1. ひと言コンセプト + メインフック
2. サービスの流れ（5ステップを図解）
3. 他との違い（比較表）
4. オプション一覧
5. 問い合わせ先

**特徴**: 提案書より簡潔。「これ見せて」とパートナーに渡せるレベルのシンプルさ。紹介者が第三者に見せることも想定。

**エージェントチーム案**: `/make-deck`スキル活用。構成設計 → 本文作成 → 図解 → 最終編集

---

### 4. セミナー企画・提案資料

**ゴール**: 見込み客を集めるセミナーの企画書 + 当日スライド。

**セミナーの方向性案**:
- タイトル例：「新規を減らしても売上が上がるサロンの仕組み」
- 対象：エステサロンオーナー（5〜20名程度の小規模セミナー）
- 時間：60〜90分
- 構成：フック → 業界データで気づき → 仕組みの考え方（全貌は出さない） → 事例 → サービス紹介 → 個別相談への誘導

**注意点**:
- noteブランドルールの「出しすぎ注意」と同じ思想。ノウハウの全貌は出さず、「もっと知りたい」と思わせる
- セミナー自体がStep1（現状診断）への入口になる設計

**エージェントチーム案**: セミナー企画設計 → スライド作成（`/make-deck`スキル活用）
