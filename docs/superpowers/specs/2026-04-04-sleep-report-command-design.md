# 睡眠解析レポート作成コマンド `/sleep-report` 設計書

## 概要

GPTs（ChatGPT有料版）で構築していた「睡眠解析・改善提案レポート」作成アシスタントを、Claude Codeのスラッシュコマンドとして再構築する。

### 背景・目的

- ChatGPT有料版の解約に伴い、Claude Codeへ移行
- 移行に伴う3つの改善目標：
  1. **内容のブラッシュアップ** — 総合スコア新設、詳細版別紙の追加
  2. **デザインのブラッシュアップ** — ダッシュボード型レイアウト、★評価、チャート類
  3. **操作の簡素化** — PC初心者スタッフでも迷わない一括添付＋自動仕分け方式

---

## 1. システムアーキテクチャ

### 実装方式

スラッシュコマンド（カスタムコマンド）として実装。

```
配置場所: ~/.claude/commands/sleep-report.md
```

### 処理フロー

```
/sleep-report（コマンド起動）
  │
  ├─ Phase 1: 受付
  │    ├─ データ一括受付
  │    ├─ 自動種類判別・仕分け
  │    ├─ チェックリスト表示
  │    └─ 確認ステップ（日数・期間・特記事項）
  │
  ├─ Phase 2: 解析
  │    ├─ 睡眠指標算出（8指標）
  │    ├─ 総合スコア算出（/100）
  │    ├─ PSS判定
  │    ├─ 生活習慣評価
  │    └─ 総評・提案生成
  │
  └─ Phase 3: レポート出力
       ├─ メインレポート（A4×2枚）HTML生成
       ├─ 詳細版（別紙読み物）HTML生成
       ├─ プレビュー表示（ブラウザで open）
       └─ スタッフ確認 → 修正対応 → 最終版PDF保存
```

### 出力先

```
~/Desktop/claude-outputs/nemunemu/sleep-report/YYYYMMDD_レポート/
  ├─ sleep-report-main.html
  ├─ sleep-report-main.pdf
  ├─ sleep-report-detail.html
  └─ sleep-report-detail.pdf
```

## 2. スタッフ操作フロー

### 操作手順

```
スタッフ                         Claude Code
───────                         ──────────
/sleep-report と入力
                                →「データを全部添付してください。
                                   画像・Excel・PDFどれでも一度にまとめてOKです」

画像・Excel・PDFをまとめて添付
                                → 自動で種類を判別し、チェックリスト表示
                                  ✅ 睡眠解析（日次）○日分
                                  ✅ 週間集計
                                  ✅ PSS・生活習慣チェック
                                  ✅ コンサルティングシート
                                  ✅ 体圧測定画像
                                  ✅ PIMAPITTA

                                →「確認です：
                                   ・睡眠解析は○日分でOKですか？
                                   ・期間は○/○〜○/○で合っていますか？
                                   ・特記事項があれば追記してください（任意）」

「OK」or「OK。特記：肩こりがひどい」
                                →「解析を開始します」
                                → 解析実行 → HTML生成 → プレビュー表示
                                →「レポートが完成しました。確認してください」

（確認して修正があれば伝える）
                                → 修正 → 再生成 → 再プレビュー
```

### GPTsからの改善ポイント

- **一括添付** — 「何をどの順番で」の迷いが消える
- **自動仕分け** — 種類判別をClaudeが自動で行う
- **チェックリスト** — 抜け漏れが一目で分かる
- **日数確認** — 必ず挟むので誤解析を防止
- **ヒアリング自動読取** — コンサルティングシートPDFから自動。特記事項のみ任意入力
- **テンプレ入力不要** — Excelへの転記作業を最小限に

## 3. データソースと受付仕様

### 受け取るデータ一覧

| # | データ | 形式 | 必須 | 備考 |
|---|---|---|---|---|
| 1 | 睡眠解析（日次 7〜20日）| 画像（スクショ）| ✅ | 専用デバイス管理画面。複数枚可 |
| 2 | 週間集計 | 画像（スクショ）| ✅ | 週画面/週平均値 |
| 3 | PSS・生活習慣チェック | Excel(.xlsx) | ✅ | 紙アンケートからの転記済み |
| 4 | コンサルティングシート | PDF | ✅ | ヒアリング内容含む。個人情報含む |
| 5 | 体圧測定結果 | 画像 | ○ | 2枚目貼付用。解析はしない |
| 6 | PIMAPITTA解析結果 | 画像 | ○ | 2枚目貼付用。解析はしない |
| 7 | 特記事項 | テキスト入力 | 任意 | 確認ステップで入力 |

### チェックリスト表示フォーマット

```
【受領チェックリスト】
1) 睡眠解析（日次）…… ✅ ○日分（期間：○/○〜○/○）
2) 週間集計 …… ✅ / ⬜ / ❓
3) PSS・生活習慣チェック …… ✅ / ⬜ / ❓
   - ネガティブ/ポジティブ/総合/リスク： ✅ / ⬜
   - 5カテゴリ点数+総合点+評価(A/B/C)： ✅ / ⬜
4) コンサルティングシート …… ✅ / ⬜ / ❓
5) 貼付用画像
   - 体圧測定 …… ✅ / ⬜
   - PIMAPITTA …… ✅ / ⬜

【確認事項】
・睡眠解析は○日分でOKですか？
・期間は○/○〜○/○で合っていますか？
・特記事項があれば追記してください（任意）
```

### 不足データ時の挙動

- 未添付の必須項目 → 「○○が不足しています。添付してください」と案内
- 画像から日付が読めない → ❓として「日数を教えてください」と確認
- 算出できない項目 → 空欄にせず「算出不可（理由）」と明記
- 推定した項目 → 必ず「推定」と明記し、根拠を1行添える

## 4. 解析ロジック

### 4.1 睡眠指標（8項目）

| # | 指標 | 算出方法 | 理想値 |
|---|---|---|---|
| 1 | 平均睡眠時間 | 全日の平均（h:mm） | 年齢別レンジ |
| 2 | 寝つき平均時間 | 下記優先順位で算出（分） | 10分前後 |
| 3 | 寝返り回数平均 | 全日の平均 | 20〜30回 |
| 4 | 活動量平均 | 1〜5スケールの平均 | 3以上 |
| 5 | 覚醒回数分布 | 3カテゴリの日数と割合 | 0〜2回が多いほど良 |
| 6 | 就寝中の向き | 4分類の%（仰向け/右/左/うつ伏せ） | — |
| 7 | 入眠の体の向き | 多い向き＋代表的遷移 | — |
| 8 | ソーシャルジェットラグ | 中央値からの最大ズレ（分） | 小さいほど良 |

### 4.2 寝つき時間の推定優先順位

```
A) 画面に「入眠時間: 00:xx」と明記 → そのまま採用（最優先）
B) 入眠までの赤線/入眠開始が読み取れる → 可能なら推定
C) 上記不可 → 寝つき評価(1〜5)から推定マッピング：
   - 5: 0〜10分 / 4: 10〜20分 / 3: 20〜30分 / 2: 30〜45分 / 1: 45分以上
   ※推定時は必ず「推定」と表記
```

- 5分以内が多い場合：「睡眠負債の可能性」コメントを添える（断定しない）

### 4.3 ソーシャルジェットラグ算出

```
1. 全日の入眠時刻・起床時刻・睡眠時間の中央値を算出
2. 各日について中央値からのズレ(分)を算出：
   ズレ = max(|入眠-中央値|, |起床-中央値|, |睡眠時間-中央値|)
3. 最大ズレの日を採用し、その値をジェットラグとして表示
4. ズレの主因（遅寝/遅起き/長寝/短寝）を短文で説明
```

※休日/平日の区分は使用しない（独自算出方式）

### 4.4 総合スコア算出（/100点）— 新規追加

| 項目 | 配点 | 算出方法 |
|---|---|---|
| 睡眠時間の充足度 | 20点 | 理想時間からの乖離で減点 |
| 寝つきの良さ | 15点 | 10分前後で満点、離れるほど減点 |
| リズムの安定性 | 20点 | ジェットラグの大きさで減点 |
| 覚醒の少なさ | 15点 | 0-2回日の割合で算出 |
| 活動量 | 10点 | 平均3以上で満点 |
| 生活習慣 | 10点 | 54点満点からの換算（低いほど良い→反転） |
| ストレス | 10点 | PSS総合からの換算（低いほど良い→反転） |

### 4.5 PSS（Perceived Stress Scale）

```
リスク判定（総合点）：
- 0〜13：低リスク
- 14〜26：中ストレス
- 27以上：高ストレス

表示項目：ネガティブ合計、ポジティブ合計、総合点、リスク
コメント：睡眠指標との"つながり"を1〜2行で説明
```

### 4.6 生活習慣チェック（54点版）

```
カテゴリ配点：
- 睡眠環境: 15 / 食事・飲料: 15 / 運動: 9 / ストレス管理: 6 / 就寝・起床: 9（合計54）

3段階評価（低いほど良い）：
- A: 19〜28（良好）
- B: 29〜38（やや問題あり）
- C: 39〜54（改善で向上が見込める）
```

### 4.7 前処理ルール

- 解析対象日数：7〜20日。欠損日は除外し「欠損◯日」と注記
- 活動量スコア：1〜5
- 覚醒回数カテゴリ：0〜2回 / 3〜5回 / 5回以上（固定）
- 寝返り適正：20〜30回（固定）
- 体の向き：睡眠中（グレー塗り範囲）のみ集計
- 時刻の日付跨ぎ：自動補正

## 5. 総評と提案の生成ルール

### 総評の統合順序

「睡眠・生活習慣・ストレス・ヒアリング主訴」を統合して作成する。

優先順：
1. 睡眠状態の分析結果（事実の要約）
2. 生活習慣の改善提案
3. ストレスへの改善提案
4. 睡眠環境への改善提案
5. 寝具提案

### 提案の構成

- **最優先3つ**（メインレポート1枚目に掲載）
  - 今すぐ取り組める・効果が大きい・主訴に刺さるもの
  - 各提案に「具体的行動」＋「理由1行」を添える
- **追加候補5つ**（メインレポート2枚目に掲載）
  - できそうな順に並べる
  - すべて「いつ・どれくらい・何を」の具体的行動で記述

### 矛盾時の対応

- 主訴と数値が矛盾する場合：主訴を先に扱い、「数値的にはここも気になるかもしれません」と併記

## 7. 安全ルール

| ルール | 詳細 |
|---|---|
| 医療行為禁止 | 医療行為・診断はしない。断定しない（「〜の可能性」「〜かもしれません」） |
| 個人情報保護 | 入力に含まれても出力へ一切再掲しない（伏せ字もしない、書かない） |
| 受付中の推測禁止 | 受付モード中は結論・推測を出さない。解析開始OK後のみ出力 |
| 平易な表現 | 読み手が知識ゼロでも分かる表現。専門用語は短い言い換えを添える |
| 矛盾対応 | 主訴を先に扱い、「数値的にはここも気になるかも」と併記 |

### 免責文（2枚目フッター・詳細版末尾に固定）

> 本レポートは睡眠状態を把握し、生活習慣や睡眠環境の見直しのヒントを提供するものです。医療行為・診断を行うものではありません。体調不良が続く場合は医療機関へご相談ください。

## 8. 運用・デプロイ

### スタッフPCへの展開

- 方式：初回コピー（将来的に共有フォルダ方式への切り替えも可）
- コピー対象：`~/.claude/commands/sleep-report.md`
- 前提：スタッフPCにClaude Codeがインストール済み＋Claudeアカウントでログイン済み

### レポートの修正フロー

1. プレビューを確認
2. 修正が必要な場合はテキストで指示（例：「提案の2番目を変えて」）
3. 再生成 → 再プレビュー
4. OKなら最終版として保存
